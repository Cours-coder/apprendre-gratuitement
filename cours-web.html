Chapitre 1 : Initiation à l’algorithmique

1. Qu’est-ce qu’un algorithme ?

Un algorithme est une suite finie d’étapes ordonnées permettant de résoudre un problème ou d’effectuer une tâche.
Il est indépendant du langage de programmation et peut être exprimé en langage naturel, pseudo-code ou diagramme.

Exemple :

          1. Prendre deux tranches de pain
          2. Ajouter du fromage
          3. Ajouter du jambon
          4. Refermer le sandwich
          5. Manger

2. Pourquoi apprendre l’algorithmique ?

  • Pour apprendre à raisonner logiquement
  • Pour résoudre efficacement des problèmes
  • Pour structurer sa pensée
  • Pour faciliter l’apprentissage de la programmation

3. Les éléments de base d’un algorithme

a) Instructions

Ce sont les actions exécutées : affecter, afficher, lire, calculer, etc.

          x ← 5
          Afficher x

b) Les structures fondamentales

i. Séquence

Les instructions sont exécutées les unes après les autres.

          a ← 2
          b ← 3
          c ← a + b
          Afficher c

ii. Condition (ou sélection)

          si âge ≥ 18 alors
             Afficher "Majeur"
          sinon
             Afficher "Mineur"

iii. Itération (boucles)

          pour i de 1 à 10 faire
             Afficher i
          fin pour

4. Variables et types de données

a) Définition d’une variable

Une variable est un nom associé à une valeur stockée en mémoire.

          nom ← "Ousmane"
          âge ← 20

b) Types de données courants

  • Entier : 1, 2, -3, 120
  • Réel : 3.14, -2.5
  • Booléen : vrai, faux
  • Caractère : 'A', 'z'
  • Chaîne : "Bonjour", "123"

5. Instructions utiles en pseudo-code

  • Lire une valeur saisie par l’utilisateur : Lire nom
  • Afficher une valeur ou un message : Afficher "Bonjour", Afficher nom
  • Affecter une valeur à une variable : x ← 10

6. Conditions logiques

On peut combiner les conditions avec :

  • ET : les deux conditions doivent être vraies
  • OU : au moins une condition doit être vraie
  • NON : inverse la condition
          si (âge > 18 ET pays = "Sénégal") alors
             Afficher "Autorisé"

7. Les boucles (itérations)

a) Boucle POUR

          pour i de 1 à 5 faire
             Afficher i
          fin pour

b) Boucle TANT QUE

          tant que n ≠ 0 faire
             Afficher n
             Lire n
          fin tant que

8. Bonnes pratiques

  • Utiliser des noms de variables explicites
  • Bien structurer son algorithme
  • Tester avec plusieurs cas
  • Commenter si nécessaire

9. Récapitulatif des notions

  • Algorithme : suite d’étapes ordonnées
  • Instruction : action de base
  • Variable : conteneur de valeur
  • Condition : permet un choix
  • Boucle : permet une répétition
  • Pseudo-code : écriture proche du langage naturel

Chapitre 2 : Initiation à la programmation

Chapitre 2 : Initiation à la programmation

1. Qu’est-ce que la programmation ?

La programmation consiste à écrire des instructions dans un langage compréhensible par un ordinateur pour qu’il exécute une tâche précise. C’est un moyen de communiquer avec la machine, pour lui dire quoi faire, comment et quand le faire.

Une suite d’instructions est appelée un programme. Ces instructions doivent respecter les règles de syntaxe du langage utilisé.

2. Langages de programmation

Il existe de nombreux langages de programmation. Certains sont basiques, d’autres très complexes, mais tous permettent d’automatiser des tâches. Exemples de langages :

  • Python – Simple et puissant, idéal pour les débutants
  • JavaScript – Utilisé pour les pages web interactives
  • C / C++ – Performants, souvent utilisés en systèmes et jeux
  • Java – Très utilisé en entreprise

3. Structure de base d’un programme

Un programme simple suit une structure logique :

  • Déclaration des variables
  • Entrée des données (via le clavier par exemple)
  • Traitement des données (calculs, conditions, boucles...)
  • Affichage des résultats

Voici un exemple en pseudo-code : 

          Début
            Lire a
            Lire b
            somme ← a + b
            Afficher somme
          Fin

4. Les variables

Une variable est un espace mémoire nommé qui contient une valeur. Elle peut changer au cours de l’exécution du programme.

Exemples :

  • entier : nombres entiers (ex : 5, -2, 100)
  • réel : nombres décimaux (ex : 3.14)
  • chaîne : texte (ex : "Bonjour")

5. Les instructions de base

5.1. Lecture

Permet de lire une donnée saisie par l’utilisateur. Exemple : 

          Lire nom

5.2. Affectation

Attribuer une valeur à une variable : 

          x ← 10

5.3. Affichage

Afficher une information à l’écran : 

          Afficher "Bienvenue"

6. Les conditions

Elles permettent de prendre une décision selon un test logique. 

          Si note ≥ 10 alors
             Afficher "Admis"
          Sinon
             Afficher "Échec"
          FinSi

7. Les boucles

Elles permettent de répéter une action plusieurs fois.

7.1. Boucle Pour

          Pour i de 1 à 5 faire
             Afficher i
          FinPour

7.2. Boucle Tant que

          Tant que x ≠ 0 faire
             Lire x
          FinTantQue

8. Fonctions et procédures (notion avancée)

Une fonction est un bloc de code qui exécute une tâche et retourne une valeur. 

          Fonction carré(n)
             retourner n * n
          FinFonction

Une procédure, elle, ne retourne rien mais peut afficher ou modifier des variables.

9. Erreurs fréquentes en programmation

  • Erreurs de syntaxe : oublier un mot-clé, une parenthèse, etc.
  • Erreurs logiques : le programme s’exécute mais ne donne pas le bon résultat
  • Erreurs d’exécution : division par zéro, variable non initialisée, etc.

10. Conseils pour bien débuter

  • Commencer par des programmes simples
  • Utiliser un langage facile comme Python
  • Tester souvent le code, ligne par ligne
  • Lire les messages d’erreur attentivement
  • Être patient et persévérant

Chapitre 4 : JavaScript

Introduction à JavaScript

JavaScript est un langage de programmation utilisé principalement pour créer des pages web interactives. Il est exécuté par le navigateur web et permet d'ajouter de la logique dynamique dans des pages HTML. JavaScript est un langage interprété, ce qui signifie qu'il est exécuté ligne par ligne, sans nécessiter de compilation préalable.

1. Syntaxe de base

JavaScript utilise une syntaxe qui ressemble à d'autres langages de programmation comme C, Java, ou Python. Les instructions JavaScript sont séparées par des points-virgules (`;`), bien que ce ne soit pas obligatoire dans tous les cas. 

            let message = "Bonjour, JavaScript !"; // Déclaration d'une variable
            console.log(message); // Affiche le message dans la console

2. Variables et types de données

En JavaScript, les variables peuvent être déclarées avec `let`, `const`, ou `var`. La différence réside dans la portée de la variable et si elle peut être modifiée après sa déclaration.

Exemples de types de données :
  • String : Chaîne de caractères
  • Number : Nombres, entiers ou à virgule flottante
  • Boolean : Vrai ou faux
  • Array : Liste de valeurs
  • Object : Objet avec des propriétés et des valeurs
            let age = 25; // Nombre
            let nom = "John"; // String
            let estEtudiant = true; // Boolean
            let fruits = ["pomme", "banane", "orange"]; // Array
            let personne = { nom: "Alice", age: 30 }; // Object

3. Structures de contrôle

JavaScript propose des structures conditionnelles et des boucles pour contrôler l'exécution du programme en fonction de certaines conditions ou pour répéter des actions.

3.1 Condition if
            let nombre = 10;
            if (nombre > 5) {
              console.log("Le nombre est plus grand que 5");
            } else {
              console.log("Le nombre est inférieur ou égal à 5");
            }
3.2 Boucle for
            for (let i = 0; i < 5; i++) {
              console.log(i); // Affiche les nombres de 0 à 4
            }

4. Fonctions

Une fonction est un bloc de code qui effectue une tâche particulière. Elle peut être appelée plusieurs fois dans un programme pour exécuter une même tâche. 

            function addition(a, b) {
              return a + b;
            }
            let resultat = addition(5, 3); // Appel de la fonction
            console.log(resultat); // Affiche 8

5. Événements et DOM

Le DOM (Document Object Model) est une interface qui permet d'interagir avec la structure HTML d'une page web. JavaScript permet de modifier le contenu, les styles, et de réagir aux événements utilisateurs.

5.1 Sélection d'éléments
            let titre = document.getElementById("monTitre");
            titre.innerHTML = "Bonjour, JavaScript !"; // Change le contenu de l'élément avec l'id "monTitre"
5.2 Ajouter un événement
            let bouton = document.getElementById("monBouton");
            bouton.addEventListener("click", function() {
              alert("Le bouton a été cliqué !");
            });

6. Conclusion

JavaScript est un langage puissant pour le développement web. Il permet d’ajouter des interactions dynamiques et est un incontournable pour tout développeur web.

Chapitre 4 : Introduction au web

Définition du Web

Le mot "Web" vient de l’anglais "World Wide Web" (WWW), qui signifie littéralement "toile d’araignée mondiale". Il s'agit d’un système hypertexte fonctionnant sur Internet, permettant la consultation de pages via des navigateurs web comme Google Chrome, Mozilla Firefox, Safari, etc.

Différence entre Internet et le Web

Beaucoup de personnes confondent le Web et Internet. Internet est l'infrastructure, c’est-à-dire l’ensemble des réseaux et des connexions. Le Web est un service qui utilise Internet pour fonctionner. Par exemple, les e-mails, les jeux en ligne, la messagerie instantanée sont d'autres services qui passent par Internet, mais qui ne font pas partie du Web.

Historique du Web

Le Web a été inventé en 1989 par Tim Berners-Lee, un chercheur au CERN (Organisation européenne pour la recherche nucléaire). Son objectif était de permettre aux scientifiques d’échanger facilement des informations. En 1991, le Web devient public. Depuis, il a connu une croissance spectaculaire, devenant un outil indispensable dans tous les domaines.

Fonctionnement du Web

Le Web fonctionne selon un principe client-serveur :

  • Le client est le navigateur utilisé par l’internaute (ex : Chrome, Firefox).
  • Le serveur est un ordinateur distant qui héberge des pages web.

Lorsqu’un utilisateur tape une adresse dans son navigateur (ex : www.google.com), le navigateur envoie une requête au serveur correspondant. Le serveur répond en envoyant le contenu demandé (souvent du HTML, CSS, JS), qui est ensuite affiché dans le navigateur.

Les technologies du Web

Le Web repose sur plusieurs technologies :

  • HTML (HyperText Markup Language) : le langage de base pour structurer une page web.
  • CSS (Cascading Style Sheets) : utilisé pour styliser et mettre en forme les pages HTML.
  • JavaScript : un langage de programmation qui permet de rendre les pages interactives.
  • Serveurs Web : logiciels comme Apache ou Nginx qui gèrent les requêtes et livrent les pages web aux clients.

URL (Uniform Resource Locator)

Une URL est l'adresse d'une ressource sur le Web. Exemple : https://www.example.com. Elle est composée de plusieurs parties :

  • Le protocole : http ou https
  • Le nom de domaine : www.example.com
  • Le chemin d'accès : /page/contact.html

Le rôle des navigateurs web

Un navigateur web est un logiciel qui permet de visiter des sites internet. Il interprète le code HTML, CSS, JavaScript envoyé par les serveurs, et l’affiche sous forme visuelle. Exemples de navigateurs : Google Chrome, Mozilla Firefox, Microsoft Edge, Safari.

Les moteurs de recherche

Un moteur de recherche est un outil qui permet de retrouver des sites ou des pages web. Il fonctionne grâce à des robots appelés crawlers qui parcourent Internet pour indexer les pages. Exemples : Google, Bing, Yahoo, DuckDuckGo.

Les types de sites web

On distingue plusieurs types de sites web :

  • Sites vitrines : présentent une entreprise ou une activité, sans fonctionnalités avancées.
  • Sites e-commerce : permettent la vente de produits ou services en ligne.
  • Blogs : plateformes de publication d’articles.
  • Réseaux sociaux : permettent l’interaction entre utilisateurs (ex : Facebook, Twitter).

Le responsive design

Avec l’essor des smartphones et tablettes, il est important que les sites s’adaptent à toutes les tailles d’écran. Le "responsive design" est une méthode de conception web qui rend les pages compatibles avec tous les supports (ordinateur, téléphone, tablette).

Les métiers du Web

Voici quelques métiers liés au développement web :

  • Développeur Front-End : conçoit l’interface visible par les utilisateurs.
  • Développeur Back-End : gère les fonctionnalités côté serveur (bases de données, sécurité, etc.).
  • Web designer : s’occupe de l’aspect graphique du site.
  • Intégrateur web : transforme les maquettes graphiques en code HTML/CSS.

Conclusion

Le Web est aujourd’hui un outil incontournable dans la vie quotidienne, que ce soit pour la communication, le travail, le commerce ou le divertissement. Sa compréhension est essentielle pour tout développeur débutant.

Chapitre 5 : HTML/CSS

Chapitre 1 : HTML et CSS

1. Introduction à HTML

HTML signifie HyperText Markup Language. C’est le langage utilisé pour créer la structure d’une page web. Il permet d’organiser le contenu : titres, paragraphes, images, liens, tableaux, formulaires, etc.

Structure de base d’un fichier HTML

Un fichier HTML commence toujours par une déclaration <!DOCTYPE html> suivie de la balise racine <html>. À l’intérieur, on trouve deux parties principales : <head> (en-tête) et <body> (corps).

Exemple de structure HTML minimale :

    <!DOCTYPE html>
    <html>
      <head>
        <title>Ma première page</title>
      </head>
      <body>
        <h1>Bonjour, monde !</h1>
        <p>Ceci est un paragraphe de texte.</p>
      </body>
    </html>

Les principales balises HTML

  • <h1> à <h6> : Titres de différents niveaux
  • <p> : Paragraphe
  • <a href=""> : Lien hypertexte
  • <img src="" alt=""> : Image
  • <ul> et <ol> : Listes
  • <div> : Conteneur générique
  • <span> : Conteneur en ligne

2. Introduction au CSS

CSS signifie Cascading Style Sheets. Il permet de styliser une page HTML, c’est-à-dire d’ajouter des couleurs, des marges, des polices, des tailles, des arrière-plans, etc.

Trois façons d’intégrer du CSS :

  • En ligne : via l’attribut style dans la balise HTML
  • Interne : avec une balise <style> dans le <head>
  • Externe : via un fichier .css lié avec <link>

Exemple CSS en ligne :

    <p style="color: red;">Texte en rouge</p>

Syntaxe CSS

Un style CSS est composé d’un sélecteur, d’accolades et de propriétés :

    p {
      color: blue;
      font-size: 16px;
    }

Principaux sélecteurs CSS

  • p : tous les paragraphes
  • .ma-classe : tous les éléments ayant cette classe
  • #mon-id : l’élément ayant cet ID
  • * : tous les éléments

Exemples de propriétés CSS

  • color : couleur du texte
  • background-color : couleur de fond
  • margin : marge extérieure
  • padding : marge intérieure
  • border : bordure
  • text-align : alignement du texte

Exemple de fichier CSS externe :

    /* style.css */
    h1 {
      color: green;
      text-align: center;
    }

Intégration dans le HTML :

    <link rel="stylesheet" href="style.css">

3. Bonnes pratiques

  • Utiliser des noms de classes clairs
  • Séparer le HTML et le CSS
  • Tester dans plusieurs navigateurs
  • Rendre la page responsive (adaptée aux écrans mobiles)

Chapitre 6 : réseaux

Cours Complet : Réseaux Informatiques

Qu'est-ce qu'un réseau informatique ?

Un réseau informatique est un ensemble d'ordinateurs et de dispositifs interconnectés permettant de partager des ressources et des informations. Les réseaux permettent aux dispositifs de communiquer entre eux grâce à des protocoles et des technologies de transmission de données.

Types de Réseaux

Réseau Local (LAN)

Un réseau local (LAN - Local Area Network) couvre une petite zone géographique comme une maison, un bureau ou un campus universitaire. Il permet aux dispositifs de partager des fichiers, des imprimantes et d'autres ressources de manière efficace et rapide.

Réseau Étendu (WAN)

Un réseau étendu (WAN - Wide Area Network) relie des réseaux locaux géographiquement dispersés. Un exemple typique de WAN est Internet, qui connecte des millions de dispositifs à travers le monde.

Réseau Métropolitain (MAN)

Un réseau métropolitain (MAN - Metropolitan Area Network) est plus grand qu'un LAN mais plus petit qu'un WAN. Il relie plusieurs LAN dans une même zone urbaine ou une ville. Un exemple pourrait être un réseau qui connecte plusieurs bureaux d'une entreprise dans une ville.

Réseau Personnel (PAN)

Un réseau personnel (PAN - Personal Area Network) est utilisé pour connecter des dispositifs personnels comme un téléphone, un ordinateur portable, une tablette, etc., dans une petite zone comme une maison ou un bureau personnel. La technologie Bluetooth est souvent utilisée pour les PAN.

Composants d'un Réseau

  • Ordinateurs et Dispositifs : Ce sont les dispositifs finaux d'un réseau, comme des ordinateurs, des serveurs, des imprimantes, etc., qui envoient et reçoivent des données.
  • Switchs : Un switch (commutateur) connecte plusieurs appareils sur un réseau local (LAN). Il permet à ces appareils de communiquer efficacement entre eux.
  • Routeurs : Un routeur connecte différents réseaux (par exemple, un réseau local à Internet) et détermine le meilleur chemin pour envoyer les données.
  • Modems : Un modem (modulateur-démodulateur) est utilisé pour connecter un réseau à Internet via une ligne téléphonique ou un câble. Il convertit les signaux numériques en signaux analogiques et vice versa.
  • Serveurs : Les serveurs fournissent des ressources et des services aux autres dispositifs du réseau, comme des fichiers, des applications ou des pages web.

Protocoles de Communication

TCP/IP

Le TCP/IP (Transmission Control Protocol / Internet Protocol) est le modèle de communication principal utilisé sur Internet. Il permet la transmission de données en garantissant leur fiabilité et leur ordre. TCP est responsable de la segmentation des données en paquets et de leur réassemblage, tandis qu'IP est responsable de l'adressage et du routage de ces paquets à travers les réseaux.

HTTP/HTTPS

Le HTTP (Hypertext Transfer Protocol) est un protocole utilisé pour la transmission de pages web. HTTPS est une version sécurisée de HTTP, utilisant un chiffrement SSL/TLS pour protéger la confidentialité des données échangées entre le client (navigateur) et le serveur.

FTP

Le FTP (File Transfer Protocol) est utilisé pour transférer des fichiers entre des ordinateurs via un réseau. Il permet de télécharger ou de téléverser des fichiers depuis et vers des serveurs.

DNS

Le DNS (Domain Name System) est un système qui traduit les noms de domaine (comme www.example.com) en adresses IP que les ordinateurs peuvent comprendre. Cela permet de naviguer sur Internet en utilisant des noms conviviaux au lieu d'adresses numériques complexes.

DHCP

Le DHCP (Dynamic Host Configuration Protocol) est un protocole qui attribue automatiquement des adresses IP aux dispositifs sur un réseau. Cela simplifie la gestion des adresses IP sur un réseau local.

Modèle OSI (Open Systems Interconnection)

Le modèle OSI est un modèle de communication en sept couches qui standardise les fonctions de communication d'un réseau. Chaque couche a un rôle spécifique et communique avec les couches adjacentes pour assurer la transmission des données.

  • Couche 1 : Physique - Gestion des supports physiques pour la transmission de données (câbles, fibre optique, ondes radio, etc.).
  • Couche 2 : Liaison de données - Assure la transmission fiable des données entre deux dispositifs directement connectés.
  • Couche 3 : Réseau - Assure le routage des paquets de données à travers le réseau, en choisissant le meilleur chemin.
  • Couche 4 : Transport - Garantit la livraison correcte et dans l'ordre des paquets de données entre l'émetteur et le récepteur.
  • Couche 5 : Session - Gère l'établissement, la gestion et la terminaison des sessions de communication entre deux dispositifs.
  • Couche 6 : Présentation - S'occupe de la présentation des données (format, compression, chiffrement). Elle veille à ce que les données soient compréhensibles par l'application réceptrice.
  • Couche 7 : Application - Cette couche permet à l'utilisateur d'interagir avec les applications réseau, comme les navigateurs web ou les clients de messagerie.

Sécurité des Réseaux

Pare-feu (Firewall)

Un pare-feu est un dispositif de sécurité qui contrôle le trafic entrant et sortant d'un réseau. Il peut être matériel ou logiciel et permet de bloquer les connexions non autorisées tout en permettant les communications légitimes.

Chiffrement

Le chiffrement est un processus qui rend les données illisibles sans une clé de déchiffrement. Il est utilisé pour protéger la confidentialité des données échangées sur des réseaux non sécurisés, comme Internet.

VPN (Virtual Private Network)

Un VPN permet de créer une connexion sécurisée entre un utilisateur et un réseau, souvent utilisé pour accéder à des ressources internes d'une entreprise ou pour naviguer anonymement sur Internet.

Contrôle d'accès

Le contrôle d'accès définit qui peut accéder à quelles ressources sur un réseau. Cela inclut des mécanismes comme les mots de passe, les cartes d'accès, ou l'authentification multi-facteurs (MFA).
